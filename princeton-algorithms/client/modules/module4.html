Iterables java.lang

In Java, an "iterable" refers to any object that can be iterated over, meaning you can traverse through its elements sequentially. The main requirement for an object to be considered iterable is that it must implement the Iterable interface . This interface declares a single method, iterator(), which returns an iterator over the elements of the iterable

  • Iterable Interface: This interface is found in the java.lang package and has a method iterator() that returns an Iterator object.
  • Iterator Interface: Objects that implement Iterator allow you to traverse through elements sequentially and typically support operations like hasNext() and next().
  • Enhanced for:Each loop : Iterates over elements of collection or array. Below iterableCollection is a collection/array that should implement Iterable Interface
    • for(elementType element : iterableCollection){}

Print prime numbers using Bag

public class PrimeNumsBag<T> implements Iterable<T>{

    public static boolean isPrime(int num){
        for(int i = 2; i <= Math.sqrt(num); i++)
            if(num % i == 0)
                return false;
            return true;
    }

    public static String printBagContents(Bag<Integer> bag){
        Iterator<Integer> itr = bag.iterator();
        String element = "";
        while(itr.hasNext()){
            element += "\n" + itr.next();
        }
        return element;
    }
    public static void main(String[] args){
        Bag<Integer> bag = new Bag<>();
        int limit = 10;

        //loop over starting from 2 till limit 
        for(int i = 2; i <= limit; i++){

            //Check if number is prime
            if(isPrime(i))
            bag.add(i);     //Adding element in bag if prime
        }
        System.out.println("The prime numbers till " + limit + ":" + printBagContents(bag));
    }
}

APIs

Bag 
public class Bag <T> implements Iterable <T>
{
    Bag()                   // create an empty bag
    void add(T t)     // add an item
    boolean isEmpty()       // is the bag empty?
    int size()              // number of items in the bag
}
Queue 
public class Queue <T> implements Iterable <T>
{
    Queue()                   // create an empty bag
    void enqueue(T t)        // void add(T t); - throws exception // void offer(T t); - return value
    boolean isEmpty()       // is the bag empty?
    T dequeue()            // T remove(); - throws exception // T poll(); - returns value
    int size()            // number of items in the queue
}

FIFO Queues

Items in queue addedd are processed in the order they were added or preserved. Any number which is enqueued is added at the end. And a number always at the start is dequeued 

public static int[] readInts(String name){
    In in = new In(name);
    Queue<Integer> queue = new Queue<>();
    while(!in.isEmpty())
        q.enqueue(in.readInt());

    int N = q.size();
    int[] a = new int[N];
    for(int i = 0; i <= N; i++)
        a[i] = a.dequeue();
    return a;
